Programming language

Seraphim

A compiled general-purpose language implemented on top of C++. Designed around a single concept of freezing — a mechanism for independently restricting the mutability of a variable's type and value. Source files use the .sp extension.

Minimalism
Minimal keywords — maximum syntactic expressiveness.
Freezing
Independent restriction of type and value mutability via a single mechanism.
Type inference
Automatic type deduction — an organic part of syntax, not an exception.
Metaprogramming
Powerful metaprogramming without redundant constructs.

Variable declaration syntax

The central syntactic construct of the language is the : operator, which separates the type and the variable name:

sp 
// explicit type
int:x = 5;
string:name = "Seraphim";

// type inference (type omitted)
:y = 10;
Difference from other languages: In Seraphim, the absence of a type to the left of : is an organic part of a unified syntax system, not an exception — unlike auto in C++ or var in Swift.

Freezing system

The key feature of the language is the ability to independently restrict the mutability of a variable's type and value. Implemented via the absolute qualifier and the :: operator.

Qualifier position

absolute:: before type — freezes the variable's type
::absolute after name — freezes the variable's value
Both together — freezes everything
sp 
int:x = 5;                             // type and value are mutable
absolute::int:y = 10;                 // type frozen
int:z::absolute = 15;                 // value frozen
absolute::int:w::absolute = 20;       // everything frozen

Four variable states

Type mutable Type frozen
Value mutable :x = 5; absolute::int:x = 5;
Value frozen :x::absolute = 5; absolute::int:x::absolute = 5;

References and pointers

References and pointers are declared between the type and the name — as part of the type-name relationship:

sp 
int:&ref = x;
int:*ptr = &x;
int:&ref2::absolute = x;
Combining a reference with value freezing means freezing the reference itself, not the value it points to.

Functions

Functions are declared using the same type:name scheme. For functions with no return value, the freed keyword is used — an analogue of void, semantically related to the concept of freeing from a type.

sp 
int:add(int:a, int:b) {
    :result = a + b;
    return result;
};

freed:greet() {
    sproute("Hello!"L:end);
};

// parameters with freezing
int:add(absolute::int:a, absolute::int:b) {
    return a + b;
};

Entry point

The entry point is the function freed:main(). To terminate the process, done() is used — a separate construct semantically different from a regular return.

sp 
freed:main() {
    done();        // exit code 0
    // done(1);   // exit code 1 — error
};

Classes

Classes are declared with class:ClassName. Instead of public and private, the keywords visible and invisible are used, reflecting accessibility in human‑readable terms.

sp 
class:Animal {
visible:
    string:name;
    freed:speak();

invisible:
    int:health = 100;
};

Control flow

To exit a loop, breakcore is used instead of the standard break:

sp 
for (...) {
    if (...) breakcore;
};

Standard library spio

Input and output are implemented via the built-in spio library, which provides two main functions.

sproute
Output data to the console with optional modifiers.
sprinp
Input data with automatic type detection.

sproute — output

sp 
sproute(expression);
sproute(expression, L:modifier);

The modifier is written adjacent to the string literal via L: — analogous to numeric suffixes in C++:

sp 
sproute("Hello, World!"L:end);    // output + newline
sproute("Hello"L:space);         // output + space
sproute("Hello"L:",");           // output + comma
sproute("Hello");               // output only
sproute(name L:end);            // variable
The prefix L: is a namespace for built-in modifiers and may be extended in the future.

sprinp — input

sp 
sprinp(variable);
sprinp(variable, buffer_size);
sprinp(variable, buffer_size, direction);

sprinp automatically considers the variable's type when parsing input. The second argument sets the buffer size in characters, the third is a boolean value determining whether cursor movement is allowed during input.

sp 
string:name;
int:age;

sprinp(name, 64);
sprinp(age, 8, true);
sprinp(string:city, 32);          // declaration directly in call

Complete program example

Save this code in a file with the .sp extension.

sp 
#include <spio>

class:Person {
visible:
    string:name;
    int:age;

    freed:greet() {
        sproute("My name is "L:space);
        sproute(name L:space);
        sproute("and I am "L:space);
        sproute(age L:space);
        sproute("years old."L:end);
    };

invisible:
    absolute::string:species::absolute = "Human";
};

int:add(absolute::int:a, absolute::int:b) {
    :result = a + b;
    return result;
};

int:main() {
    Person:p;
    sprinp(p.name, 64);
    sprinp(p.age, 4);
    p.greet();

    absolute::int:x = 5;
    int:y::absolute = 10;
    :sum = add(x, y);

    sproute("Sum: "L:space);
    sproute(sum L:end);

    done();
};
